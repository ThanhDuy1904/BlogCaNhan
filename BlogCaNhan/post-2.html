Lập Trình Hướng Đối Tượng (OOP) Trong Java

1 OOP là gì?

Lập trình hướng đối tượng (Object-Oriented Programming - OOP) là một phương pháp lập trình dựa trên khái niệm "đối tượng", có thể chứa dữ liệu dưới dạng thuộc tính (attributes/fields) và code dưới dạng phương thức (methods). Java là ngôn ngữ OOP thuần túy, mọi thứ trong Java đều là đối tượng (trừ các kiểu dữ liệu nguyên thủy).

OOP giúp tổ chức code theo cách gần gũi với thế giới thực, mỗi đối tượng đại diện cho một thực thể có thuộc tính và hành vi riêng. Ví dụ: một đối tượng "Xe hơi" có thuộc tính như màu sắc, tốc độ, và hành vi như chạy, dừng, rẽ.

Khái niệm cơ bản

  • Class (Lớp): Là bản thiết kế (blueprint) để tạo ra các đối tượng
  • Object (Đối tượng): Là thể hiện (instance) cụ thể của một class
  • Attribute (Thuộc tính): Là dữ liệu của đối tượng
  • Method (Phương thức): Là hành vi của đối tượng

2 Encapsulation (Đóng gói)

Encapsulation là việc gói dữ liệu (attributes) và các phương thức xử lý dữ liệu đó vào một đơn vị duy nhất gọi là class. Dữ liệu được ẩn đi (private) và chỉ có thể truy cập thông qua các phương thức public (getter và setter).

Lợi ích của Encapsulation

  • Bảo vệ dữ liệu khỏi truy cập trái phép
  • Kiểm soát cách dữ liệu được đọc và ghi
  • Dễ dàng thay đổi implementation mà không ảnh hưởng code bên ngoài
  • Tăng tính bảo mật và linh hoạt

Ví dụ về Encapsulation

public class Student {
    // Private attributes - ẩn dữ liệu
    private String name;
    private int age;
    private double gpa;
    
    // Constructor
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }
    
    // Getter methods - để đọc dữ liệu
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
    
    public double getGpa() {
        return gpa;
    }
    
    // Setter methods - để ghi dữ liệu với validation
    public void setName(String name) {
        if (name != null && !name.isEmpty()) {
            this.name = name;
        }
    }
    
    public void setAge(int age) {
        if (age > 0 && age < 150) {
            this.age = age;
        }
    }
    
    public void setGpa(double gpa) {
        if (gpa >= 0.0 && gpa <= 4.0) {
            this.gpa = gpa;
        }
    }
    
    // Method để hiển thị thông tin
    public void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("GPA: " + gpa);
    }
}

3 Inheritance (Kế thừa)

Inheritance cho phép một class (class con - subclass) kế thừa các thuộc tính và phương thức từ class khác (class cha - superclass). Điều này giúp tái sử dụng code và tạo mối quan hệ phân cấp giữa các class.

Các loại Inheritance trong Java

  • Single Inheritance: Một class chỉ kế thừa từ một class cha
  • Multilevel Inheritance: Class A → Class B → Class C
  • Hierarchical Inheritance: Nhiều class con kế thừa từ một class cha

Lưu ý: Java không hỗ trợ Multiple Inheritance (một class kế thừa từ nhiều class) để tránh Diamond Problem. Thay vào đó, sử dụng Interface.

Ví dụ về Inheritance

// Class cha (Superclass)
public class Person {
    protected String name;
    protected int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void introduce() {
        System.out.println("Hello, I'm " + name);
    }
}

// Class con (Subclass) kế thừa từ Person
public class Student extends Person {
    private String studentId;
    private double gpa;
    
    public Student(String name, int age, String studentId, double gpa) {
        super(name, age); // Gọi constructor của class cha
        this.studentId = studentId;
        this.gpa = gpa;
    }
    
    // Override method từ class cha
    @Override
    public void introduce() {
        System.out.println("Hi, I'm " + name + ", student ID: " + studentId);
    }
    
    public void study() {
        System.out.println(name + " is studying...");
    }
}

// Class con khác
public class Teacher extends Person {
    private String subject;
    private int experience;
    
    public Teacher(String name, int age, String subject, int experience) {
        super(name, age);
        this.subject = subject;
        this.experience = experience;
    }
    
    @Override
    public void introduce() {
        System.out.println("Hello, I'm " + name + ", I teach " + subject);
    }
    
    public void teach() {
        System.out.println(name + " is teaching " + subject);
    }
}

4 Polymorphism (Đa hình)

Polymorphism cho phép một phương thức có thể có nhiều hình thái khác nhau. Trong Java, có hai loại Polymorphism: Compile-time (Static) và Runtime (Dynamic).

Method Overloading (Compile-time Polymorphism)

Nhiều methods cùng tên nhưng khác tham số trong cùng một class.

public class Calculator {
    // Overloading - cùng tên khác tham số
    public int add(int a, int b) {
        return a + b;
    }
    
    public double add(double a, double b) {
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    public String add(String a, String b) {
        return a + b;
    }
}

Method Overriding (Runtime Polymorphism)

Class con ghi đè (override) method của class cha.

public class Animal {
    public void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Woof! Woof!");
    }
}

public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meow! Meow!");
    }
}

// Sử dụng
public class Main {
    public static void main(String[] args) {
        Animal myAnimal;
        
        myAnimal = new Dog();
        myAnimal.makeSound(); // Output: Woof! Woof!
        
        myAnimal = new Cat();
        myAnimal.makeSound(); // Output: Meow! Meow!
    }
}

5 Abstraction (Trừu tượng)

Abstraction là việc ẩn đi chi tiết triển khai và chỉ hiển thị tính năng thiết yếu. Trong Java, Abstraction được thực hiện thông qua Abstract Class và Interface.

Abstract Class

Là class không thể khởi tạo trực tiếp, có thể chứa abstract methods và concrete methods.

public abstract class Shape {
    protected String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    // Abstract method - không có implementation
    public abstract double calculateArea();
    public abstract double calculatePerimeter();
    
    // Concrete method - có implementation
    public void displayColor() {
        System.out.println("Color: " + color);
    }
}

public class Circle extends Shape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}

public class Rectangle extends Shape {
    private double width;
    private double height;
    
    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    @Override
    public double calculateArea() {
        return width * height;
    }
    
    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
}

Interface

Là contract hoàn toàn trừu tượng, tất cả methods đều abstract (trước Java 8).

public interface Drawable {
    void draw(); // abstract by default
    void resize(double factor);
}

public interface Movable {
    void move(int x, int y);
}

// Class có thể implement nhiều interfaces
public class Square implements Drawable, Movable {
    private int x, y;
    private double side;
    
    @Override
    public void draw() {
        System.out.println("Drawing square at (" + x + ", " + y + ")");
    }
    
    @Override
    public void resize(double factor) {
        side *= factor;
        System.out.println("Resized to: " + side);
    }
    
    @Override
    public void move(int newX, int newY) {
        x = newX;
        y = newY;
        System.out.println("Moved to: (" + x + ", " + y + ")");
    }
}

6 Lợi ích của OOP

  • Tái sử dụng code: Thông qua Inheritance và Polymorphism
  • Dễ bảo trì: Code được tổ chức rõ ràng, dễ tìm và sửa lỗi
  • Linh hoạt: Dễ dàng mở rộng và thêm tính năng mới
  • Bảo mật: Encapsulation bảo vệ dữ liệu
  • Dễ hiểu: Mô hình hóa gần với thế giới thực
  • Giảm độ phức tạp: Chia nhỏ vấn đề thành các đối tượng
